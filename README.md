# Оранжерея — помощник по уходу за растениями

Справочник растений, личная коллекция, избранное, напоминания о поливе и
пересадке, и обмен растениями между пользователями с чатом. Данные хранятся
на сервере в SQLite и синхронизируются между устройствами по токену устройства
(аккаунт с паролем не нужен).

## Состав (всё в одной папке)

```
index.html         страница
styles.css         стили
data.js            встроенный справочник (запасной, если сервер недоступен)
storage.js         обмен данными с сервером
notifications.js   расчёт задач по уходу
app.js             логика интерфейса, обмен и чат
server.js          Express: REST API + раздача статики
db.js              подключение к SQLite (встроенный node:sqlite)
schema.sql         схема БД + наполнение справочника
package.json       зависимости (только express) и версия Node
.node-version      версия Node для Render (24)
render.yaml        деплой на Render одним файлом
```

## Запуск локально

Нужен Node 22.5+ или 24. В папке проекта:

```bash
npm install
node server.js
```

Открыть **http://localhost:3000** (именно этот адрес, а не файл — фронтенд
работает через сервер). Терминал держать открытым; остановка — Ctrl+C.

При старте Node пишет `ExperimentalWarning: SQLite is an experimental feature`
— это нормально, встроенный SQLite помечен экспериментальным.

## Обмен растениями (как пользоваться)

Вкладка **«Обмен»** разделена на две части: сверху **ваши объявления**, снизу
**чужие**.

- В строке профиля укажите имя — оно видно собеседникам в чате.
- **«Разместить растение»** — выберите растение из справочника (по желанию),
  заголовок, описание и что хотите взамен.
- **Чужое объявление** → кнопка **«Написать владельцу»** открывает ваш личный
  диалог с автором (по одному чату на каждое объявление).
- **Ваше объявление** → кнопка **«Отклики»** показывает список всех, кто вам
  написал; с каждым — **отдельный чат**. Там же статус (Открыто /
  Зарезервировано / Закрыто) и удаление объявления.
- Сообщения в открытом чате обновляются автоматически каждые 3 секунды.

Диалог виден только двум участникам (владельцу объявления и откликнувшемуся) —
посторонние доступа к переписке не имеют.

Чтобы проверить обмен вдвоём, откройте сайт в двух разных браузерах или в
обычном окне и в режиме инкогнито — это два разных «устройства».

## Деплой на Render

Вариант с файлом `render.yaml` (проще всего):
1. Залейте папку в репозиторий GitHub.
2. В Render: **New → Blueprint**, укажите репозиторий. Render прочитает
   `render.yaml` и создаст веб-сервис (build: `npm install`, start: `npm start`).
3. После сборки откроется адрес вида `https://ваш-сервис.onrender.com`.

Вариант вручную: **New → Web Service**, репозиторий, Build Command `npm install`,
Start Command `npm start`, переменная `NODE_VERSION = 24`.

### Важно про сохранность данных

На бесплатном плане Render диск **эфемерный**: файл `plants.db` обнуляется при
перезапуске и передеплое (справочник наполнится заново, но объявления, чаты и
личные списки сбросятся). Для демонстрации этого обычно достаточно.

Чтобы данные сохранялись постоянно, подключите **Persistent Disk** (доступен на
платных инстансах): в `render.yaml` раскомментируйте блок `disk` и переменную
`DATA_DIR=/var/data`. Код уже это поддерживает — путь к базе берётся из
переменной `DATA_DIR`.

## Точки API (кратко)

Профиль: `GET/PATCH /api/me`.
Справочник: `GET /api/plants`, `GET /api/plants/:id`.
Избранное: `GET /api/favorites`, `POST|DELETE /api/favorites/:plantId`.
Личный список: `GET|POST /api/my-plants`, `PATCH|DELETE /api/my-plants/:id`,
`POST /api/my-plants/:id/water`.
Обмен: `GET|POST /api/exchange`, `PATCH|DELETE /api/exchange/:id`,
`GET /api/exchange/:id/conversations` (отклики на моё объявление),
`POST /api/exchange/:id/open` (открыть диалог с владельцем),
`GET|POST /api/conversations/:cid/messages`, `GET /api/conversations/:cid`.
Все запросы к `/api/*` шлют заголовок `X-Device-Token`.
